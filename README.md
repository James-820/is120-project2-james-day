# James Day - IS120 Project 2

### Initial Ideas

I want to make this with mobile in mind first, and then later I'll add in a bunch of stuff for larger screens. I signed up to demo Figma and Neumorphism, so I decided to try making the design in Figma first, then replicating that with my code. I really like a lot of the Neumorphism design elements, so I'll probably include buttons and things like that as much as I can. This will help me maintain a clean minimalist look to my website.

## Overview

### Description

This is a portfolio website for myself. This is something that I could include a link to on a business card or include with a resume. This website is something that could be added to, tweaked, and improved over time. I'd like it to include professional experience, personal projects, skills, and even some fun facts about me (like the "Warhammer 40,000 Army" section).

### Learning

I learned a few things about CSS animations in order to get the spinners and typing animations to work properly. At first, I had the typing animation end with the "cursor" sitting and blinking at the end, but for some reason, that bothered me. So I changed the cursor to only appear while the typing was going, then disappear so it's not bugging people while they're trying to read. It makes it feel a little bit less like an actual text box, but that is worth the trade in my opinion. Another fun fact about the typing animation is that I originally saw it on YouTube Shorts, of all places.

I also learned about the `animation-play-state` property, and how to use the `paused` and `running` keywords to get the spinners to keep their position while the mouse isn't hovering on them.

### Problems

One of the biggest problems I ran into was trying to get my `grid` layout in the `main` element all correct. Getting things laid out well for mobile was actually pretty easy, all I did was put everything into one column. However, trying to move that structure around a little for wider screens was a bit of an ordeal. I kept the basic column structure of the page, but moved the content blocks around to make room for some spinners to give a little life to the page. When I tried dividing the page for the first time, I must have done something wrong with the `grid-template-areas`, because everything was on top of each other, even the footer was rendering right below the header. Luckily, after speaking with my good friend Claude, I figured out what was going wrong, and finally got a better-looking layout.

### Dev Tools

Some things that went really well that I did include were a lot of the dev tools that were presented in class. I used Coolors to extract a color palette from my profile picture so that I could use them on my website. I didn't use all of them, and I kind of wish I had found a way to include more of those colors. I also really liked the Feather Icons, as you can see, I used a lot of them all over the page. They were helpful in both the contact info section and for describing the other sections in my website. The clean look of those really brought out a lot in the website. I also really liked the Neumorphism style that I presented on, and tried it out as buttons at first. I changed later to making them into content sections, as I couldn't get the buttons dialed in the right way.

### Next Time

I would do a lot of things differently next time. That's not to say that I don't like what I made, I actually am really proud of where I got to. That being said, next time I would really put a lot of thought and planning as to what I wanted my website to look like and plan it out on paper beforehand. I tried using Figma to do a mockup of the website, but I decided that it was a little too tedious. Planning would have given me a little more direction as to what I could have included in the website.

All things considered, I am very pleased with the way this turned out. I think it's a very clean aesthetic that helps things flow well. I'm probably going to go back and change things to be even better next time.
